# JavaScript সহজ প্রশ্নোত্তর

## 1) null এবং undefined এর পার্থক্য কী?

`undefined` এটি একটি ডিফল্ট অবস্থা। যখন আপনি একটি ভ্যারিয়েবল ডিক্লেয়ার করেন কিন্তু কোনো মান সেট করেন না, তখন জাভাস্ক্রিপ্ট নিজে থেকেই তাকে undefined মান দেয়   

``` js
let x;
console.log(x); // undefined
```

`null` এটি একটি ইচ্ছাকৃত অবস্থা। যখন আপনি বুঝাতে চান যে একটি ভ্যারিয়েবল বর্তমানে "খালি" বা এতে "কোনো অবজেক্ট নেই", তখন আপনি নিজে হাতে তাকে null অ্যাসাইন করেন

``` js
let y = null;
console.log(y); // null
```

## 2) map() কী কাজে লাগে? forEach() থেকে পার্থক্য কী?
জাভাস্ক্রিপ্টে map() এবং forEach() উভয়ই অ্যারের প্রতিটি এলিমেন্টের ওপর লুপ চালানোর জন্য ব্যবহৃত হয়, তবে এদের কাজ করার পদ্ধতি এবং উদ্দেশ্য সম্পূর্ণ ভিন্ন

`map()` মূলত একটি অ্যারের ডেটা পরিবর্তন বা Transform করার কাজে ব্যবহৃত হয় [১.১.২, ১.৩.৬]। এটি অ্যারের প্রতিটি উপাদানের ওপর একটি ফাংশন চালায় এবং সেই ফাংশন থেকে পাওয়া রেজাল্ট দিয়ে একটি নতুন অ্যারে তৈরি করে দেয় [১.১.৪, ১.২.৯]। মূল (Original) অ্যারেটি এতে অপরিবর্তিত থাকে [১.১.৬, ১.৪.৩]। 

`forEach()` যখন আপনার শুধু লুপ চালিয়ে কোনো কাজ করতে হবে (যেমন: console.log() করা, ডেটাবেসে ডেটা সেভ করা বা DOM পরিবর্তন করা) এবং আপনার কোনো নতুন অ্যারের প্রয়োজন নেই


যদি আপনি লুপ শেষে একটি রেজাল্ট অ্যারে ফেরত পেতে চান, তবে map() ব্যবহার করুন; আর যদি শুধু কাজ সেরে ফেলতে চান তবে forEach() ব্যবহার করুন

## 4) async/await কেন দরকার?

জাভাস্ক্রিপ্টে `Asynchronous` কাজগুলোকে (যেমন: API থেকে ডেটা আনা বা ফাইল পড়া) আরও সহজে এবং পরিষ্কারভাবে হ্যান্ডেল করার জন্য `async/await` ব্যবহার করা হয়। এটি মূলত Promises ব্যবহারের একটি আধুনিক এবং সহজ পদ্ধতি।

`async/await` ব্যবহার করলে ব্রাউজারের ডিবাগার ঠিক সেই লাইনেই থামে যেখানে কাজ চলছে, যা then() ব্লকের ভেতরে করা কিছুটা কঠিন।

## 5) Scope কী?

জাভাস্ক্রিপ্টে Scope বলতে বোঝায় কোডের কোন অংশে কোন ভ্যারিয়েবল বা ফাংশন ব্যবহার করা যাবে (Access) আর কোথায় যাবে না। অর্থাৎ, এটি একটি ভ্যারিয়েবলের "সীমানা" বা "এলাকা" নির্ধারণ করে।

### Global Scope

যদি কোনো ভ্যারিয়েবল কোনো ফাংশন বা ব্লকের বাইরে ডিক্লেয়ার করা হয়, তবে সেটি গ্লোবাল স্কোপে থাকে। এই ভ্যারিয়েবল কোডের যেকোনো জায়গা থেকে অ্যাক্সেস করা যায়।


### Function Scope

একটি ফাংশনের ভেতরে ডিক্লেয়ার করা ভ্যারিয়েবল শুধুমাত্র ওই ফাংশনের ভেতরেই কাজ করে। বাইরে থেকে একে কল করা সম্ভব নয়। var, let, এবং const সবাই ফাংশন স্কোপ মেনে চলে।


### Block Scope

ES6-এ আসা let এবং const ব্লক স্কোপ মেনে চলে। ব্লক বলতে মূলত থার্ড ব্র্যাকেট { ... } এর ভেতরকার অংশকে বোঝায় (যেমন if, for লুপ)। তবে মনে রাখবেন, var ব্লক স্কোপ মানে না।
